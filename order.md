### [Phase 1] 4대 경쟁사 통합 크롤링 스크립트 생성
로그인 보안과 각기 다른 DOM 구조를 처리하기 위한 파이썬 스크립트 제작용입니다.

> **Prompt (Input to GPT-OSS):**
> "Python과 Playwright를 사용하여 아래 4개 의약품 B2B몰의 '이벤트/기획전' 데이터를 수집하는 스크립트를 작성해줘.
> 
> **대상 사이트:**
> 1. HMP몰 (hmpmall.co.kr)
> 2. 바로팜 (baropharm.com)
> 3. 새로팜 (saeropharm.com)
> 4. 플랫폼팜 (platpharm.co.kr)
> 
> **요구사항:**
> 1. **Page Object Pattern:** 각 사이트별로 `Scraper` 클래스를 분리하여 유지보수가 쉽게 구성해줘.
> 2. **로그인 자동화:** `id`, `pw`를 환경변수(`os.getenv`)에서 읽어와 로그인 세션을 유지해줘. (플랫폼팜의 로그인 경로 확인 필요)
> 3. **추출 필드:** `mall_name`, `event_title`, `benefit_summary` (할인/적립 등), `duration` (기간), `target` (신규/전체), `detail_url`.
> 4. **에러 핸들링:** 특정 사이트가 점검 중이거나 로그인이 실패해도 다른 사이트 수집은 계속 진행되도록 `try-except` 처리해줘.
> 5. **출력:** 최종 데이터를 `competitor_raw.csv` 파일로 저장해줘."

### [Phase 2] 데이터 분석 및 마크다운 리포트 생성
수집된 데이터를 시니어 보고용 마크다운으로 가공하는 단계입니다.

> **Prompt (Input to GPT-OSS):**
> "앞서 생성된 `competitor_raw.csv`를 읽어서 시니어 마케터가 한눈에 파악할 수 있는 `weekly_report.md`를 생성해줘. 
> 
> **구성 내용:**
> 1. **Summary 테이블:** 사이트별로 현재 진행 중인 이벤트 개수와 가장 파격적인 혜택(최고 할인율 등)을 요약해줘.
> 2. **이벤트 리스트:** 4개 사이트 통합 리스트를 작성하되, '종료 임박'순으로 정렬해줘.
> 3. **태깅 로직:** 혜택 내용에 '현금성/리베이트' 소지가 있는 키워드(예: 골드바, 백화점 상품권, 여행권)가 포함되면 [❗약사법 주의] 태그를 붙여줘.
> 4. **출력 형식:** 나중에 JSX에서 파싱하기 좋게 섹션 제목은 `#`, 테이블 형식은 표준 마크다운 문법을 준수해줘."

### [Phase 3] CSV 데이터를 JSX 대시보드로 변환
최종적으로 대리님이 웹에 게시할 React(JSX) 컴포넌트를 만드는 과정입니다.

> **Prompt (Input to GPT-OSS):**
> "수집된 `competitor_raw.csv` 데이터를 화면에 렌더링할 `CompetitorDashboard.jsx` 파일을 작성해줘. Tailwind CSS를 사용해.
> 
> **기능 요구사항:**
> 1. **Grid Layout:** 4개 경쟁사별 섹션을 나누거나, 전체 리스트를 카드(Card) 형태로 보여줘.
> 2. **필터/검색:** 상단에 '몰 별 보기' 필터와 '혜택 유형별(적립/할인/사은품)' 필터를 구현해줘.
> 3. **데이터 매핑:** JSON 데이터를 `map()` 함수로 돌려 카드를 생성하고, 기간이 만료된 이벤트는 회색 처리를 하거나 목록에서 제외해줘.
> 4. **JSX 스타일링:** 플랫폼팜처럼 모던하고 깔끔한 UI를 위해 각 카드에는 몰 이름별로 고유 색상 배지(예: HMP-블루, 바로팜-그린 등)를 달아줘.
> 5. **JSON 변환 스크립트:** CSV를 이 컴포넌트가 바로 읽을 수 있는 `data.json`으로 변환하는 간단한 파이썬 코드도 함께 줘."
